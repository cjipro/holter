## Key architectural moves this evening

**1. Diagnosis methodology fills a gap Value/Risk alone couldn't.**
Hussain's framing — "is this a support problem (deploy AI) or a journey problem (fix the journey)?" — turned out to be a missing layer in the decision flow. The Value × Risk 2×2 tells you how big the prize is and how exposed you are; it doesn't tell you whether the intervention addresses the actual problem locus. Diagnosis runs first and can OVERRIDE the 2×2 (JOURNEY_PROBLEM → "fix the journey" verb regardless of Action tier; INCONCLUSIVE → NEEDS_MORE_DATA regardless of how appealing the cell looks).

**2. The pack-declaration → methodology-vocabulary contract is real.**
PULSE-104 backfilled `severity_class: high|medium|low` per pack. PULSE-106 reads that field and maps to P0/P1/P2 for methodology consumption. The engine never hardcodes severity by signature — it reads what the pack declared. This is the contract that lets new packs land without engine changes.

**3. Two-stage trust on Chronicle is load-bearing.**
All 10 seed CHR-friction entries ship `verification_status: pending_human_review`. The matcher fails closed. Risk methodology consumes zero Chronicle precedents in the seed-batch state. HOL-9's Chronicle panel renders this transparently with a curator-handoff note — the design discipline (engine state is honest in the briefing, never silently degrades) is now a reusable pattern across HOL-9 and HOL-11.

**4. Colour palette unified across briefing surfaces.**
`_DIAGNOSIS_COLORS`, `_RISK_COLORS`, `_VALUE_COLORS`, `_ACTION_COLORS` in `render_mil_briefing.py`. Severity-ascending baseline (grey → teal → amber); top tiers diverge by meaning (REGULATORY-FLAG red, COMMERCIAL-OPPORTUNITY green). Test-asserted: `_RISK_COLORS` keys must equal PULSE-99 rubric tier_words; `_VALUE_COLORS` keys must equal PULSE-101 methodology tier_words. Catches future enum drift before it ships silently as grey-fallback badges.
